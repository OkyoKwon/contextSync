# Multi-perspective AI Evaluation Plan

> AI 활용 능력(AI Literacy & Competency) 다관점 평가 지표

---

## Gemini 평가 기준

AI 활용 능력을 평가하는 지표는 단순히 '도구를 얼마나 잘 다루는가'를 넘어, **윤리적 이해**, **비판적 사고**, 그리고 **실무 적용 능력**이 조화를 이루어야 합니다.

크게 **4가지 영역**으로 구분한 평가 기준 프레임워크입니다.

### 1. 기술적 이해 및 활용 (Technical Proficiency)

AI의 작동 원리를 이해하고 도구를 능숙하게 다루는 단계입니다.

| 하위 항목               | 설명                                                                                                              |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **도구 선택 능력**      | 문제 해결을 위해 최적의 AI 모델(텍스트, 이미지, 코드, 데이터 분석 등)을 선택할 수 있는가?                         |
| **프롬프트 엔지니어링** | 페르소나 설정, 제약 조건 부여, Few-shot prompting 등 고도화된 기법을 사용하여 원하는 결과값을 이끌어낼 수 있는가? |
| **멀티모달 활용**       | 텍스트뿐만 아니라 이미지, 음성, 파일 분석 등 다양한 입력 방식을 복합적으로 활용할 수 있는가?                      |

### 2. 비판적 사고 및 검증 (Critical Thinking & Verification)

AI가 생성한 결과물을 맹신하지 않고 검토하며 수정하는 능력입니다.

| 하위 항목                   | 설명                                                                                      |
| --------------------------- | ----------------------------------------------------------------------------------------- |
| **할루시네이션(환각) 식별** | AI 결과물의 사실 여부를 교차 검증(Cross-check)하고 오류를 찾아낼 수 있는가?               |
| **편향성 감지**             | AI 출력물에 내재된 사회적, 문화적 편향성을 인지하고 이를 조정할 수 있는가?                |
| **결과물 최적화**           | AI의 초안을 바탕으로 본인의 전문성을 더해 최종 결과물의 품질을 높이는 편집 능력이 있는가? |

### 3. 워크플로우 통합 및 문제 해결 (Integration & Problem Solving)

AI를 실제 업무 프로세스에 녹여내어 생산성을 높이는 단계입니다.

| 하위 항목             | 설명                                                                         |
| --------------------- | ---------------------------------------------------------------------------- |
| **업무 자동화**       | 반복적인 태스크를 AI를 통해 자동화하거나 소요 시간을 획기적으로 단축했는가?  |
| **해결책 설계**       | 복잡한 문제를 단계별로 구조화하여 AI에게 지시하고 해결책을 도출할 수 있는가? |
| **도구 커스터마이징** | GPTs 제작이나 API 연동 등 본인의 목적에 맞게 AI 환경을 구축할 수 있는가?     |

### 4. 윤리 및 보안 의식 (Ethics & Security)

AI 시대에 필수적인 책임감 있는 사용 능력입니다.

| 하위 항목          | 설명                                                                                  |
| ------------------ | ------------------------------------------------------------------------------------- |
| **데이터 보안**    | 민감한 내부 정보나 개인정보를 AI 학습에 유출하지 않도록 보안 가이드라인을 준수하는가? |
| **저작권 및 인용** | AI 생성물의 저작권 이슈를 이해하고, 적절한 출처 표기 및 윤리적 기준을 지키는가?       |

### 숙련도별 레벨 정의

| 레벨   | 명칭                | 주요 특징                                                    |
| ------ | ------------------- | ------------------------------------------------------------ |
| **L1** | 입문자 (Awareness)  | AI의 존재를 알고 기본적인 질문(Chat)을 던질 수 있음          |
| **L2** | 사용자 (User)       | 일상적 업무에 AI를 활용하며, 간단한 프롬프트 수정이 가능함   |
| **L3** | 숙련가 (Advanced)   | 복합적인 프롬프트를 작성하고, AI 결과물을 비판적으로 검증함  |
| **L4** | 전략가 (Strategist) | 업무 프로세스 전체에 AI를 이식하여 생산성을 2배 이상 높임    |
| **L5** | 선도자 (Innovator)  | 새로운 AI 활용법을 창안하거나 조직 내 AI 가이드라인을 수립함 |

## ChatGPT 평가 기준

AI 활용 능력을 평가할 때 단순히 "AI를 쓸 줄 아는가"가 아니라, **문제 정의 → 활용 방식 → 결과 품질 → 운영/개선 능력**까지 전반을 구조적으로 보는 것이 중요합니다.

**7가지 영역**으로 구분한 평가 기준 프레임워크입니다.

### 1. 문제 정의 능력 (Problem Framing) — 15%

AI를 쓰기 전에 **무엇을 해결하려는지 명확히 정의하는 능력**

| 하위 항목             | 설명                                           |
| --------------------- | ---------------------------------------------- |
| **문제 구체화**       | 문제를 구체적으로 설명할 수 있는가             |
| **AI 적용 범위 판단** | AI로 해결 가능한 영역과 아닌 영역을 구분하는가 |
| **목표 설정**         | 목표(Output)가 명확하게 설정되어 있는가        |

| 레벨 | 기준                                 |
| ---- | ------------------------------------ |
| Lv1  | 막연한 요청 ("이거 좀 잘되게 해줘")  |
| Lv2  | 문제는 설명 가능하지만 목표가 불명확 |
| Lv3  | 목표/제약조건까지 명확히 정의        |
| Lv4  | AI 적용 범위 + 기대 결과까지 구조화  |

### 2. 프롬프트 설계 능력 (Prompt Engineering) — 20%

AI에게 **정확한 결과를 얻기 위한 입력 설계 능력**

| 하위 항목         | 설명                                |
| ----------------- | ----------------------------------- |
| **지시 명확성**   | 명확한 지시 / 역할 부여 가능 여부   |
| **컨텍스트 설계** | 컨텍스트, 예시, 포맷 정의 여부      |
| **반복 개선**     | 반복 개선(Iterative prompting) 능력 |

| 레벨 | 기준                                     |
| ---- | ---------------------------------------- |
| Lv1  | 단순 질문형 입력                         |
| Lv2  | 조건 일부 포함                           |
| Lv3  | 구조화된 프롬프트 (역할/형식 포함)       |
| Lv4  | Few-shot / Chain-of-thought 등 고급 활용 |

### 3. 결과 해석 및 검증 능력 (Output Validation) — 20%

AI 결과를 **비판적으로 검토하고 수정하는 능력**

| 하위 항목              | 설명                                       |
| ---------------------- | ------------------------------------------ |
| **정확성 판단**        | 결과의 정확성 판단                         |
| **Hallucination 구분** | hallucination 구분 가능 여부               |
| **결과 다듬기**        | 결과를 실제 활용 가능 수준으로 다듬는 능력 |

| 레벨 | 기준                        |
| ---- | --------------------------- |
| Lv1  | 결과를 그대로 사용          |
| Lv2  | 오류 일부 인지              |
| Lv3  | 검증 + 수정 가능            |
| Lv4  | 교차 검증 및 품질 기준 적용 |

### 4. AI 활용 효율성 (Efficiency) — 15%

AI를 통해 **시간/비용 대비 효율적으로 결과를 만드는 능력**

| 하위 항목        | 설명                  |
| ---------------- | --------------------- |
| **자동화**       | 반복 작업 자동화 여부 |
| **모델/툴 선택** | 적절한 모델/툴 선택   |
| **호출 최적화**  | 불필요한 호출 최소화  |

| 레벨 | 기준               |
| ---- | ------------------ |
| Lv1  | 단순 사용          |
| Lv2  | 일부 효율 개선     |
| Lv3  | 작업 흐름 최적화   |
| Lv4  | 자동화 및 시스템화 |

### 5. 도구 및 생태계 활용 능력 (Tooling) — 10%

AI 외에도 **관련 도구를 함께 활용하는 능력** (API, 데이터베이스, 자동화 툴 등)

| 레벨 | 기준                   |
| ---- | ---------------------- |
| Lv1  | 단일 AI 툴만 사용      |
| Lv2  | 여러 툴 병행 사용      |
| Lv3  | API 연동 가능          |
| Lv4  | end-to-end 시스템 구축 |

### 6. 문제 해결 및 응용 능력 (Adaptability) — 10%

AI를 다양한 상황에 맞게 **응용하는 능력**

| 레벨 | 기준                         |
| ---- | ---------------------------- |
| Lv1  | 정형화된 사용만 가능         |
| Lv2  | 유사 문제에 적용             |
| Lv3  | 새로운 문제에도 확장 가능    |
| Lv4  | 전략적 활용 (업무 혁신 수준) |

### 7. 협업 및 커뮤니케이션 (Collaboration) — 10%

AI 활용 결과를 **다른 사람과 공유/협업하는 능력**

| 레벨 | 기준                        |
| ---- | --------------------------- |
| Lv1  | 개인 사용                   |
| Lv2  | 결과 공유                   |
| Lv3  | 팀 내 활용 가능 형태로 정리 |
| Lv4  | 조직 내 표준화 기여         |

### 종합 가중치

| 항목                               | 가중치 |
| ---------------------------------- | ------ |
| 문제 정의 (Problem Framing)        | 15%    |
| 프롬프트 설계 (Prompt Engineering) | 20%    |
| 결과 검증 (Output Validation)      | 20%    |
| 효율성 (Efficiency)                | 15%    |
| 도구 활용 (Tooling)                | 10%    |
| 응용 능력 (Adaptability)           | 10%    |
| 협업 (Collaboration)               | 10%    |

### 숙련도별 레벨 정의

| 레벨    | 명칭         | 주요 특징             |
| ------- | ------------ | --------------------- |
| **Lv1** | Beginner     | 단순 사용             |
| **Lv2** | Intermediate | 실무 활용 가능        |
| **Lv3** | Advanced     | 효율적/전략적 활용    |
| **Lv4** | Expert       | 시스템화 및 조직 확산 |

### 핵심 포인트

이 지표의 핵심은 단순히 "AI를 잘 쓰는가"가 아니라, **"AI를 통해 문제를 더 잘 해결하는가"** 입니다.

## 구현 기획: Multi-perspective AI Evaluation

### 개요

"Run Evaluation" 버튼을 한 번 누르면 **Claude / ChatGPT / Gemini** 3개 관점에서 동시에 평가가 실행됩니다. 각 관점은 자체 고유 차원으로 평가하며, API는 모두 **Claude API**를 사용합니다 (Claude에게 각 관점의 평가자 역할을 부여).

### 핵심 설계 원칙

| 원칙                         | 설명                                                          |
| ---------------------------- | ------------------------------------------------------------- |
| **Single Trigger**           | 사용자는 한 번만 클릭 — 백엔드에서 3개 평가를 병렬 실행       |
| **Independent Perspectives** | 각 관점은 완전히 독립된 차원/가중치/레벨 체계를 가짐          |
| **Unified API**              | Claude API 하나로 3개 관점 모두 처리 (시스템 프롬프트만 변경) |
| **Graceful Degradation**     | 일부 관점이 실패해도 성공한 관점의 결과는 정상 표시           |

---

### 3개 관점 요약

| 관점              | 차원 수              | 레벨 체계       | 평가 철학                                 |
| ----------------- | -------------------- | --------------- | ----------------------------------------- |
| **Claude** (기존) | 5개                  | 5단계 (0-100)   | 프롬프트 품질 + AI 고유 기능 활용 중심    |
| **ChatGPT**       | 7개                  | 4단계 (Lv1-Lv4) | 문제 정의 → 활용 → 검증 → 협업 파이프라인 |
| **Gemini**        | 4개 영역 (11개 하위) | 5단계 (L1-L5)   | 윤리/비판적 사고/실무 적용의 조화         |

### 백엔드 변경사항

#### DB 스키마 변경 (새 마이그레이션)

**ai_evaluations 테이블 변경:**

```
ALTER TABLE ai_evaluations ADD COLUMN perspective VARCHAR(20) NOT NULL DEFAULT 'claude';
-- perspective: 'claude' | 'chatgpt' | 'gemini'

ALTER TABLE ai_evaluations ADD COLUMN evaluation_group_id UUID;
-- 같은 트리거로 생성된 3개 평가를 묶는 그룹 ID
```

**새 인덱스:**

```
CREATE INDEX idx_ai_evaluations_group ON ai_evaluations(evaluation_group_id);
```

**변경 효과:**

- 기존 evaluation 레코드는 `perspective = 'claude'`, `evaluation_group_id = NULL`로 하위 호환
- 한 번의 트리거로 3개의 evaluation 행이 생성됨 (같은 `evaluation_group_id`)
- 각 evaluation은 독립된 dimensions/evidence를 가짐

#### 새로운 타입 정의 (packages/shared)

```typescript
// 관점 타입
type EvaluationPerspective = 'claude' | 'chatgpt' | 'gemini';

// 관점별 차원 정의
const CHATGPT_DIMENSIONS = [
  'problem_framing',
  'prompt_engineering',
  'output_validation',
  'efficiency',
  'tooling',
  'adaptability',
  'collaboration',
] as const;

const GEMINI_DIMENSIONS = [
  'technical_proficiency',
  'critical_thinking',
  'integration_problem_solving',
  'ethics_security',
] as const;

// ChatGPT 가중치
const CHATGPT_DIMENSION_WEIGHTS = {
  problem_framing: 0.15,
  prompt_engineering: 0.2,
  output_validation: 0.2,
  efficiency: 0.15,
  tooling: 0.1,
  adaptability: 0.1,
  collaboration: 0.1,
};

// Gemini 가중치 (4개 영역 균등 + 윤리 약간 낮게)
const GEMINI_DIMENSION_WEIGHTS = {
  technical_proficiency: 0.3,
  critical_thinking: 0.25,
  integration_problem_solving: 0.25,
  ethics_security: 0.2,
};

// ChatGPT 레벨 (4단계)
const CHATGPT_TIERS = {
  beginner: { min: 0, max: 25 },
  intermediate: { min: 26, max: 50 },
  advanced: { min: 51, max: 75 },
  expert: { min: 76, max: 100 },
};

// Gemini 레벨 (5단계)
const GEMINI_TIERS = {
  awareness: { min: 0, max: 20 },
  user: { min: 21, max: 40 },
  advanced: { min: 41, max: 60 },
  strategist: { min: 61, max: 80 },
  innovator: { min: 81, max: 100 },
};

// 그룹 응답 타입
interface EvaluationGroupResult {
  groupId: string;
  claude: AiEvaluationWithDetails | null;
  chatgpt: AiEvaluationWithDetails | null;
  gemini: AiEvaluationWithDetails | null;
}
```

#### 서비스 로직 변경

**triggerEvaluation 흐름 (변경 후):**

```
1. 기존 검증 (권한, 쿨다운, 최소 메시지 수)
2. 메시지 샘플링 (1회만 — 3개 관점이 동일 데이터 사용)
3. evaluation_group_id 생성 (UUID)
4. 3개 evaluation 레코드 생성 (status: pending)
5. Promise.allSettled로 3개 Claude API 호출 병렬 실행:
   - analyzeWithClaudePerspective(messages)   → 기존 시스템 프롬프트
   - analyzeWithChatGPTPerspective(messages)   → ChatGPT 관점 시스템 프롬프트
   - analyzeWithGeminiPerspective(messages)    → Gemini 관점 시스템 프롬프트
6. 각 결과를 독립적으로 저장 (성공/실패 개별 처리)
7. 실패한 관점은 status: failed, 성공한 관점은 status: completed
```

**핵심:** `Promise.allSettled` 사용으로 하나가 실패해도 나머지는 정상 완료

#### 새 API 엔드포인트

| Method | Endpoint                        | 설명                                    |
| ------ | ------------------------------- | --------------------------------------- |
| GET    | `/ai-evaluation/group/:groupId` | 그룹 ID로 3개 관점 결과 한 번에 조회    |
| GET    | `/ai-evaluation/latest`         | 기존 — `perspective` 쿼리 파라미터 추가 |

기존 엔드포인트는 하위 호환 유지 (perspective 미지정 시 claude 기본값).

### 에러 처리 & 엣지 케이스

| 시나리오                 | 처리 방식                                                                    |
| ------------------------ | ---------------------------------------------------------------------------- |
| 3개 중 1개만 실패        | 성공한 2개는 정상 표시, 실패한 탭에 "평가 실패" 메시지 + 재시도 버튼         |
| 3개 모두 실패            | 전체 evaluation group을 failed 처리, 에러 메시지 표시                        |
| API 토큰 부족            | 첫 번째 호출에서 429 감지 시 나머지 호출 취소 (비용 절약)                    |
| 기존 데이터 마이그레이션 | 기존 evaluation은 `perspective='claude'`, `group_id=NULL`로 유지 — 하위 호환 |

```typescript
export async function triggerEvaluation(db, projectId, userId, input, apiKey) {
  // 1. 기존 검증 (권한, 최소 메시지 수)

  // 2. 쿨다운 체크 변경: evaluation_group_id 기준
  //    - 가장 최근 completed된 group의 completed_at 확인
  //    - group_id가 NULL인 기존 레코드도 호환

  // 3. 동시 실행 가드 변경: group 단위
  //    - 해당 유저에게 진행 중인 다른 group이 있으면 차단
  //    - 같은 group 내 pending은 정상

  // 4. 메시지 샘플링 (1회)
  const sampledMessages = sampleMessages(userMessages);

  // 5. group_id 생성
  const groupId = crypto.randomUUID();

  // 6. 3개 evaluation 레코드 생성 (각각 perspective 지정)
  const evaluations = await Promise.all(
    EVALUATION_PERSPECTIVES.map((perspective) =>
      createEvaluation(db, { ...baseData, perspective, evaluationGroupId: groupId }),
    ),
  );

  // 7. 3개 분석 병렬 실행 (100ms 간격으로 시작 — rate limit 방지)
  const analysisPromises = EVALUATION_PERSPECTIVES.map((perspective, i) =>
    delay(i * 100).then(() =>
      analyzeAndSavePerspective(db, evaluations[i], perspective, sampledMessages, apiKey),
    ),
  );
  await Promise.allSettled(analysisPromises);

  // 8. 그룹 결과 반환
  return { groupId, evaluationIds: evaluations.map((e) => e.id) };
}

// 개별 관점 분석 + 저장 함수
async function analyzeAndSavePerspective(db, evaluation, perspective, messages, apiKey) {
  try {
    await updateEvaluation(db, evaluation.id, { status: 'analyzing' });

    const result = await analyzeEvaluation(apiKey, messages, sessionCount, perspective);

    // 차원/증거 저장 (기존 로직과 동일, 차원 목록만 perspective별)
    await saveDimensionsAndEvidence(db, evaluation.id, result, perspective);

    // 가중 점수 계산 (perspective별 가중치 사용)
    const weights = PERSPECTIVE_WEIGHTS[perspective];
    const overallScore = calculateOverallScore(result.dimensions, weights);
    const tier = determineProficiencyTier(overallScore, perspective);

    // 완료 업데이트
    await updateEvaluation(db, evaluation.id, {
      status: 'completed',
      overallScore,
      proficiencyTier: tier,
      improvementSummary: result.improvementSummary,
      completedAt: new Date().toISOString(),
    });
  } catch (error) {
    await updateEvaluation(db, evaluation.id, {
      status: 'failed',
      errorMessage: error.message,
    });
    // throw하지 않음 — allSettled이므로 다른 관점에 영향 없음
  }
}
```
