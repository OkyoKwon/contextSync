# Governance

## Project Roles

### Maintainers

Maintainers have full commit access and are responsible for:

- Reviewing and merging pull requests
- Triaging issues and setting priorities
- Making architectural decisions
- Managing releases

Current maintainers:

- [@OkyoKwon](https://github.com/OkyoKwon) (project lead)

### Contributors

Anyone who submits a pull request, reports an issue, or participates in discussions is a valued contributor.

## Decision Making

- **Minor changes** (bug fixes, small features, documentation): A single maintainer approval is sufficient.
- **Significant changes** (new modules, architectural changes, dependency additions): Discussion in a GitHub Issue or Discussion before implementation. Maintainer consensus required.
- **Breaking changes**: Must be discussed in a GitHub Issue with a migration plan. Require explicit maintainer approval.

## Becoming a Maintainer

Active contributors who demonstrate sustained, high-quality contributions may be invited to become maintainers. Criteria include:

- Consistent contributions over 3+ months
- Demonstrated understanding of the codebase and architecture
- Constructive participation in code reviews and discussions
- Alignment with project goals and conventions

Maintainer invitations are made by existing maintainers through consensus.

## Code of Conduct

All participants are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md).
